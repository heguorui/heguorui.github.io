【iOS】【UINavigationController】UINavigationController使用笔记

UINavigationController是iOS中，极其重要一个控件之一，几乎在所有的具有View切换、导航控制中，都是必不可少的、不可替代的控件。
这篇文章实际上不是对UINavigationController使用的特别介绍，而是我在使用中遇到的一些坑，仅做记录，同时给其他使用这个控件的人，提供一些思路，如果有错误，还请指正。

UINavigationController基本使用方法

UINavigationController是一种界面导航控制器，以栈方式管理界面，所以其管理的内容就是UIViewController。当你需要进行多种界面导航时，UINavigationController能够提供的功能有：
跳转到新界面、并能够返回
跳转到新界面、并能够指定返回的界面
返回前一界面
返回指定界面
使用时，通过push、pop两种操作，就能够实现最基本的界面跳转控制。

UINavigationController的navigationBar用法

UINavigationController顶部具有一个控件，为UINavigationBar控件。这个控件实际上是一个导航条的管理控件，你可以单独使用，但意义不大，最好跟着UINavigationController结合起来使用。
UINavigationBar的使用方式与UINavigationController极其相似，因为，它也是一种栈管理模式，管理着导航栏的内容，其内容就是UINavigationItem。
UINavigationBar是一个UIView，同样，UINavigationItem也是一个UIView。

UINavigationBar 样式处理（自定义、透明处理等）

UINavigationBar的透明处理方案有很多。

特定样式：快捷方便

UINavigationBar提供了导航条样式设置功能：setBarStyle:，样式主要由以下种类：
typedef NS_ENUM(NSInteger, UIBarStyle) {
UIBarStyleDefault = 0,
UIBarStyleBlack = 1,
UIBarStyleBlackOpaque = 1, // Deprecated. Use UIBarStyleBlack
UIBarStyleBlackTranslucent = 2, // Deprecated. Use UIBarStyleBlack and set the translucent property to YES
};
但其中，后两个被废弃了。所以，你只能设置默认和黑色风格。
如果想要透明化处理，还有另外一个方法可以设置透明开关：setTranslucent:
设置YES为透明化，设置NO为不透明化。
但这样比较局限，很多的APP还需要自定义风格之类的处理。

自定义背景

setBackgroundImage:forBarMetrics:

这个方法提供了背景颜色设置，可以给定UIImage，设置背景色。
需要注意的是，你要兼顾竖屏、横屏，即便你只支持某一种，你也要知道forBarMetrics:参数的给定：
UIBarMetricsDefault：竖屏时的处理
UIBarMetricsLandscapePhone：横屏时的处理
